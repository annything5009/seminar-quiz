# เกมควิซสำหรับงานสัมนา 🎉

เกมควิซสไตล์ Kahoot รองรับผู้เล่นได้ ~250 คนพร้อมกัน เล่นผ่านลิงก์ ไม่ต้องล็อกอิน
ทายเร็ว+ถูก ได้คะแนนเยอะ มีเฉพาะโฮสต์เท่านั้นที่เห็นคะแนนสูงสุด 3 อันดับ

## วิธีเล่น
1. โฮสต์เปิดหน้า **Host** (ต้องใส่รหัสโฮสต์ที่ปรากฏตอนรันเซิร์ฟเวอร์)
2. ผู้เข้าร่วมเปิดลิงก์ **Player** บนมือถือ ใส่แค่ชื่อ กดเข้าร่วม (ไม่ต้องสมัคร/ล็อกอิน)
3. โฮสต์กด "เริ่มเกม" → คำถามจะขึ้นทีละข้อพร้อมตัวจับเวลา
4. ผู้เล่นแตะคำตอบ ยิ่งตอบเร็วและถูก ยิ่งได้คะแนนเยอะ
5. หลังหมดเวลา (หรือทุกคนตอบครบ) หน้าจอโฮสต์จะโชว์คำตอบที่ถูก + คะแนนสูงสุด 3 อันดับ (ผู้เล่นเห็นแค่คะแนนตัวเอง)
6. โฮสต์กด "ข้อถัดไป" ไปเรื่อย ๆ จนจบ

คำถามตัวอย่าง 10 ข้อ (ความรู้ทั่วไป) อยู่ในไฟล์ `questions.json` — แก้ไข/เพิ่มคำถามของคุณเองได้เลย ตามรูปแบบเดิม:
```json
{
  "question": "คำถาม...",
  "choices": ["ตัวเลือก1", "ตัวเลือก2", "ตัวเลือก3", "ตัวเลือก4"],
  "correctIndex": 0,
  "timeLimit": 15
}
```

## ทดสอบในเครื่องตัวเอง
```
npm install
npm start
```
เทอร์มินัลจะโชว์:
```
Host key: xxxxxxxx
Player link:  http://localhost:3000/player.html
Host link:    http://localhost:3000/host.html?key=xxxxxxxx
```

## นำไปใช้จริงกับคน 250 คน (ต้องมีลิงก์สาธารณะ)
รันในเครื่องตัวเองอย่างเดียวจะมีแค่คนในวง Wi-Fi เดียวกันเข้าได้ ถ้าต้องการให้ผู้เข้าร่วมทุกคนเปิดลิงก์จากมือถือตัวเองได้ทุกที่ ให้ deploy ขึ้นเว็บฟรีตัวใดตัวหนึ่ง (รองรับ WebSocket ทั้งคู่):

**ตัวเลือกที่แนะนำ: Render.com (ฟรี)**
1. อัปโหลดโฟลเดอร์นี้ขึ้น GitHub repo (หรือใช้ Render's "Deploy from Git")
2. ไปที่ render.com → New → Web Service → เชื่อม repo
3. ตั้งค่า Build command: `npm install`, Start command: `npm start`
4. เพิ่ม Environment Variable ชื่อ `HOST_KEY` = ตั้งรหัสเองที่จำง่าย (ไม่งั้นรหัสจะสุ่มใหม่ทุกครั้งที่รีสตาร์ท)
5. เมื่อ deploy เสร็จจะได้ลิงก์ เช่น `https://your-app.onrender.com`
   - ลิงก์ผู้เล่น: `https://your-app.onrender.com/player.html`
   - ลิงก์โฮสต์: `https://your-app.onrender.com/host.html?key=รหัสที่ตั้งไว้`

**หมายเหตุสำคัญสำหรับงานจริง:**
- แผนฟรีของ Render จะ "หลับ" เมื่อไม่มีคนใช้งานนาน ๆ ให้เข้าลิงก์ทดสอบล่วงหน้าก่อนงานสัก 10-15 นาทีเพื่อปลุกเซิร์ฟเวอร์
- ถ้ากังวลเรื่องความเสถียรกับคนจำนวนมาก แนะนำอัปเกรดเป็นแผนเสียเงินราคาถูก (Render Starter หรือ Railway) ระหว่างวันงาน
- แจกลิงก์ผู้เล่นผ่าน QR code จะสะดวกกว่าให้พิมพ์ URL เอง (ใช้เว็บสร้าง QR ฟรีจากลิงก์ player.html)

## โครงสร้างไฟล์
- `server.js` — เซิร์ฟเวอร์เกม (Node.js + Socket.io)
- `questions.json` — คำถามทั้งหมด (แก้ไขได้)
- `public/player.html` — หน้าจอผู้เล่น
- `public/host.html` — หน้าจอโฮสต์ (ควบคุมเกม + เห็นคะแนนสูงสุด 3 อันดับ)
